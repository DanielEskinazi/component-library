# Speech to Text

## Basic Example

```tsx live
function SpeechDemo() {
  const [transcript, setTranscript] = useState('');
  
  return (
    <div>
      <SpeechToText 
        onTranscript={setTranscript}
        continuous={true}
        lang="en-US"
      />
      {transcript && (
        <div className="mt-4 p-4 bg-gray-100 rounded">
          <strong>You said:</strong> {transcript}
        </div>
      )}
    </div>
  );
}
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onTranscript` | `(text: string) => void` | - | Callback when transcript updates |
| `onError` | `(error: Error) => void` | - | Error handler |
| `onStateChange` | `(state: SpeechRecognitionState) => void` | - | State change handler |
| `continuous` | `boolean` | `false` | Enable continuous recognition |
| `interimResults` | `boolean` | `true` | Show interim results |
| `lang` | `string` | Browser default | Recognition language |
| `showTranscript` | `boolean` | `true` | Show transcript UI |
| `autoStart` | `boolean` | `false` | Start automatically |
| `className` | `string` | - | Additional CSS classes |

## Browser Permissions

The component requires microphone access. Users will be prompted to grant permission on first use:

```tsx
// Check permission status
const checkMicrophonePermission = async () => {
  const result = await navigator.permissions.query({ name: 'microphone' });
  console.log(result.state); // 'granted', 'denied', or 'prompt'
};
```

## Performance Considerations

- The component uses the Web Speech API which runs in a separate thread
- No impact on main thread performance
- Automatic cleanup of resources when component unmounts
- Efficient event handling with debouncing
